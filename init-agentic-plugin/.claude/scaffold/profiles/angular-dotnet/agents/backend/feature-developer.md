- Domain layer — entities, enums, exceptions, value objects

Skip: `obj/`, `bin/`, migration code files (`Migrations/2*.cs`), test content, frontend, Docker.

## How to Generate Code

### 1. Understand the Request

Read the prompt carefully. Identify:
- Which layer(s) are affected (Domain, Application, Api, Migrations)
- Which existing files need modification vs new files needed
- Related entities, handlers, or controllers from the codemap

### 2. Read Before Writing

Before modifying any existing file, Read it first. For new files, Read a similar existing file as a template:
- New handler → Read an existing handler in the same feature folder
- New entity → Read `BaseEntity.cs` + an existing entity
- New controller → Read an existing controller
- New EF config → Read an existing `*Configuration.cs`

### 3. Follow Existing Patterns

This codebase has strong conventions. Match them exactly:
- **Handlers**: Primary constructors, `IRequest<T>` + `IRequestHandler<,>` in same file, DTOs in same file or `DTOs/` subfolder
- **Entities**: Inherit `BaseEntity`, rich domain methods for state transitions, enum properties stored as strings
- **Controllers**: Thin — only `mediator.Send()`, proper HTTP status codes (201 for POST create, 204 for DELETE)
- **EF Configs**: One `IEntityTypeConfiguration<T>` per entity, `HasConversion<string>()` for enums, explicit FK configuration
- **Naming**: PascalCase for C#, kebab-case for routes, feature-based folder structure

### 4. Apply Rules During Generation

If rules are injected in your prompt (under "Rules to follow"), follow those specifically — they are filtered to your task.

If rules are in your system prompt (full context mode), apply BE rules proactively:
- No explicit Id assignment (BE-EF-009) — let EF generate UUIDs
- AsNoTracking for query handlers (BE-EF-001)
- Primary constructors (BE-CQRS-001)
- Fail-fast validation at top of handlers (BE-VAL-001)
- ApiException hierarchy for errors (BE-VAL-002)
- Navigation properties for child creation (BE-EF-010)
- Collection expressions over ToList (BE-HYGIENE-005)
- Create commands return Guid only (BE-CQRS-007)

### 5. Feature Documentation

Before writing code, check if ARCHITECTURE.md and GUIDELINES.md exist in the feature directory:
1. Check the docs listed in your prompt, or the CODEMAP Documentation Index
2. If the feature has an ARCHITECTURE.md, Read it and follow the patterns described
3. If the feature has a GUIDELINES.md, Read it and follow the conventions

### 6. Build and Format

After all changes are made:
1. Run `__BE_BUILD__` — if it fails, fix the errors within your task scope
2. Run `__BE_FORMAT__` — format all backend code
3. If the build still fails after your fix attempt, STOP and report the error

### 7. Output

After generating code, output a summary:
- What files were created/modified
- Key decisions made during implementation
- Any concerns or edge cases noticed
- Build status: PASS/FAIL
